# 통통 디펜스 (TongTong Defence)

> **통통 디펜스: 핀볼 마스터** 1스테이지 모작 — PurpleCow 채용 과제
> Unity 6000.3.10 · Android(세로) · 개발 기간 5일

몬스터를 향해 볼을 쏘아 튕기며 싸우는 핀볼 디펜스입니다. 원작 1스테이지의 핵심 루프
(발사 → 물리 반사 전투 → 레벨업 3택지 → 스킬 성장 → 웨이브 20 → 결과)를 재현했습니다.

---

## 1. 실행 방법

### APK (권장)
`Builds/TongTongDefence.apk`를 Android 기기에 설치 (Min SDK 25 / 세로 고정)

### Unity 에디터
1. Unity **6000.3.10** 으로 프로젝트 열기
2. `Assets/Scenes/SampleScene.unity` 열기 → **Play**
3. 노치 대응 확인은 Game 뷰 → **Device Simulator** 권장

### 조작
- **드래그(터치/마우스)**: 조준 — 점선 경로 + 끝점 레티클로 반사 궤적 미리보기
- 발사는 자동 (보유한 볼이 회수되는 대로 재발사)
- 우상단 **⏸**: 일시정지 (보유 스킬 확인)

### APK 직접 빌드
에디터 메뉴 **Tools → Build Android APK** — 패키지명/세로 고정/백엔드까지 스크립트가
강제 설정하므로 클릭 한 번으로 동일한 제출본이 나옵니다 (`Assets/Editor/BuildScript.cs`).

## 2. 주요 구현 내용

### 필수 요건 구현 매핑

| 요건 | 구현 |
|---|---|
| 핀볼 로직 | 물리 반사(PhysicsMaterial2D 탄성 1) + 충돌 시 속력 재정규화. 노멀볼 8, 치명타 1.5배 |
| 웨이브 20개 | **행 컨베이어** (원작 실플레이 관찰로 재설계): 시작 5행 일괄 → 한 칸 내려올 때마다 다음 행. 행 구성은 CSV 패턴(내용 17행+간격 7행, 매판 동일·결정적). 클리어 = 행 소진+전멸 |
| 로그라이크 3택지 | 레벨업(5/10/15…킬) 시 게임 정지 + 카드 3장 — 중복 없음·신규 Lv1·보유는 +1레벨·만렙 제외·만석 시 업그레이드만 |
| 스킬 10종 × 3레벨 | 액티브 볼 5종(파이어/아이스/레이저/고스트/클러스터) + 패시브 5종(양철심장/거울/단검 2종/성냥) |
| 결과 + 재시작 | Stage Clear(남은 체력 %)/Fail(진행도 %) 팝업 + 1스테이지 재시작 |

### 아키텍처 하이라이트

```
순수 C# 코어 (TongTong.Core — 엔진 무의존 어셈블리, EditMode 테스트 35개)
  스킬 테이블 파서 · 데미지 계산기 · 드래프트 규칙 · 레벨/체력 · 볼 인벤토리
스킬 "행동" 분리 (Effects/)
  IOnHitEffect + 스킬당 클래스 1개 — 새 스킬 추가 = 파일 추가 (OCP)
매니저 계층
  GameManager(상태 머신) ← 프로퍼티 주입(싱글톤 회피) — Field/Ball/Monster/Wave/Skill/Player/Stats/UI
  StatsManager = 집계 단일 창구: 모든 데미지에 "소스 태그"를 실어 이벤트로 수집 (전투 정보 창의 기반)
  매니저 간 통신은 C# 이벤트 (프리팹 인스턴스는 매니저를 모름 — 값 주입/이벤트 중계만)
데이터 주도
  스킬 수치·설명·아이콘(SkillTable.csv) + 스테이지 행 패턴(StagePattern.csv) 전부 CSV — 기획자가 코드 없이 수정
  엄격 검증 파서 — 스킬 표는 오염 행을 실제로 잡아낸 이력, 스테이지 패턴은 멀티셀 점유 정합까지 검증
필드 지오메트리 = 배경 그림이 SSOT
  격자/벽/스폰 전부 "배경 스프라이트 bounds 내 비율"에서 파생 — 화면비가 어떻든 그림과 어긋날 수 없는 구조
발사 모델 = "볼 인벤토리" (원작 분석으로 재설계)
  보유 볼 각각이 개체 — 먼저 회수된 볼부터 재발사 (쿨다운이 아니라 회수가 발사를 만든다)
전부 오브젝트 풀링
  볼/몬스터/파편/데미지 팝업 — 대량 전투 프레임 대비 사전 워밍 포함
```

### 원작 재현 디테일

- 스토어 스크린샷·실플레이를 **관찰노트로 문서화**하고 구현의 SSOT로 사용 (미관찰은 [가정] 표기 후 재관찰로 보정)
- 조준: 점 나열 점선 + 끊긴 링/빨간 점 레티클 (스프라이트 픽셀 제작), 각도 ±75° 제한
- 캐릭터: 몸/머리/지팡이 파츠 분리 — 조준 방향 응시(목 피벗·기울기 제한) + 좌우 미러링
- 상단 HUD(스테이지명·진행도·레벨 게이지)/일시정지(보유 스킬 4+2 슬롯)/데미지 플로터/몬스터 체력바
- 도달 몬스터: 3초 대기 후 플레이어에게 돌진·1회 타격·소멸 (즉사 아닌 체력전, HP 300)
- **전투 정보 창**: 일시정지에서 진입 — 볼·스킬별 [아이콘 · ◆레벨 · 누적 데미지 · 최대 대비 비율 바 · DPS].
  성냥 폭발 같은 부가 피해도 소스별 귀속 (원작 스크린샷 대조로 확인한 사양)
- 레벨 게이지: 레벨업 시 "꽉 채우는 연출 → 선택 창 → 0부터" 시퀀스, 게이지류 전부 부드러운 보간

## 3. 추가 구현 (가산점 대응)

- **상태이상 시스템**: 화상(스택형 도트) / 냉동(감속+받는 피해 증가) — 풀 재사용 안전 초기화
- **스킬 전종 실동작**: 레이저 행 일괄 피해(격자 쿼리) / 고스트 관통(레이어+센서) / 클러스터 파편볼 / 성냥 연쇄 폭발 / 단검 전·후면 치명타(적당 1회)
- **테스트 자동화**: 순수 코어 규칙 42개 (드래프트 규칙·데미지 공식·회수 순서·전투 집계·행 패턴 검증 등 수동 재현 불가 영역)
- **멀티셀 몬스터**: 사슴 1×2(세로)·돌벌레 2×1(가로) — 행 패턴 CSV가 점유 칸(`-`)까지 검증,
  전용 블록 스프라이트로 비율 왜곡 없이 배치, 콜라이더만 점유 크기로 확장
- **UI 세이프 에어리어 + 3구역 레이아웃** (TopBar/Content/BottomBar) — 노치 기기 대응, TextMeshPro 한글
- **에디터 도구 제작**: UI/캐릭터 파츠/팝업 프리팹 1회 생성 메뉴, 원버튼 APK 빌드 (`Assets/Editor/`)

## 4. AI 활용

**Claude Code를 개발 전 과정의 페어 프로그래머로 사용했습니다.** 다만 다음 원칙으로 운용했습니다:

- **사이클 강제**: 태스크 등록 → 설계 검사(SOLID/Unity 패턴 체크리스트) → 승인 → 구현 → Play 검증 → 커밋. AI가 승인 없이 구현하지 않도록 프로젝트 규약(CLAUDE.md)으로 통제
- **원작 관찰이 우선**: AI의 추측은 [가정] 태그로 격리하고, 원작 재관찰로 확정/폐기 (발사 모델을 관찰에 맞춰 2회 재설계한 이력)
- **이해가 결과물의 일부**: 일괄 생성(오토 모드) 작업은 코드 워크스루를 거쳐 병합 — 모든 코드를 직접 설명할 수 있는 상태 유지
- 트러블슈팅도 함께: 물리 콜백 중 콜라이더 제거로 인한 볼 정지, 풀 재사용 텔레포트, 대량 처치 프레임의 생성 폭풍 등을 진단 → 근본 수정 + 안전망의 이중 구조로 해결

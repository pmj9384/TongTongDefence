# 원작 관찰노트 — 통통 디펜스: 핀볼 마스터

> 원작 플레이/스토어 스크린샷에서 확인한 **외부 사실**의 단일 저장소. 코드/설계가 이 문서를 근거로 삼는다.
> 새 관찰이 기존 항목과 충돌하면 이 문서를 정정하고 날짜를 남긴다 (실사례: 노멀볼 소멸 → 튕김 정정).

## 필드/화면 구성

- 화면 3단: 상단 UI 밴드(일시정지·스테이지명·웨이브 게이지) / 중앙 플레이 판 / 하단 슈터+게이지
- 판 = 화면 폭의 ~83%, 격자 **가로 9열 × 세로 13행**, 셀은 거의 정사각
- 화면 가장자리는 어두운 비네트 (검정 여백이 자연스러움)
- 지급 배경 에셋(2048² 정사각)은 **세로 원화를 눌러 담은 압축본** — 세로 stretch가 비율 복원 (스토어 스크린샷과 파란 꽃/덩굴/뿌리 구성 일치로 확인, 2026-07-03)

## 몬스터/블록

- 몬스터는 **블록(돌 받침) 위에 앉아 한 몸으로** 하강·소멸
- 블록 = 격자 1칸 꽉 채움. 점유: Fluffy 1×1, Spider 1×1, ForestDeer 1×2, StoneBug 2×1
- **2×2 블록은 보스용** → 과제 범위 밖 (보스 제외 항목)
- 이동 = **연속 등속 하강** (칸 단위 스냅 아님) — 3회 뒤집힌 끝에 확정 (2026-07-03)
- 스폰: 웨이브 전원 **한 번에**, 매판 **동일 배치**(결정적), 종류 혼합. 드랍 시 그림자 연출
- **스폰 시작 행 = 판 상단에서 한 행 아래 (2026-07-06 확정)** — 상단 벽과 첫 줄 사이 완충 행 존재. 이 간격 덕에 스폰이 상단의 볼과 겹쳐도 볼이 벽 밖으로 밀려나지 않음 (실기기 버그의 원작식 해법)
- HP 종류별 차등 여부 불명 → 설정 가능하게만 (hpMultiplier)
- **스크린샷의 몬스터 위 숫자 = 데미지 플로터** (타격 순간 뜨는 데미지 텍스트, 빨간 숫자는 강조 표시로 추정 — 치명타?). **몬스터 HP는 맞았을 때부터 블록에 체력바로 표시** (2026-07-05 유저 정정 — "HP 숫자 표시"로 오독했던 것)

## 볼/전투

- **볼은 블록에 맞으면 데미지 주고 튕긴다 — 소멸하지 않음** (2026-07-04 재관찰로 정정. 초기 기록 "노멀볼 히트 시 소멸"은 오기록)
- 히트박스 = 블록 사각형 (몬스터 스프라이트 모양 아님). 블록의 존재 이유 = 볼을 사각형으로 튕기게
- 볼 퇴장 경로: 바닥(뿌리지대)에 튕김 → 슈터에게 귀환 → 소멸
- 발사: 쿨다운 자동 발사 + 포인트 투 슛 조준 (real-time, 턴제 아님)

## 플레이어/진행

- **플레이어 체력 존재** (슈터 아래 게이지, 예: 300) — 몬스터 바닥 도달 = 즉시 GameOver 아님, HP 감소 (2026-07-04)
- **도달 몬스터의 거동 (2026-07-05 재관찰로 정정)**: 일반 몬스터는 바닥 도달 시 **약 3초 머문 뒤** 플레이어에게 회수(돌진)되며 **부딪혀 1회 데미지 주고 소멸** (충돌 시 플레이어에게 데미지 텍스트 + 피 이펙트). 3초 대기 동안 처치 가능. ~~멈춰서 반복 공격~~은 **보스 전용 거동**(가만히 서서 공격 모션+지속 데미지) — 보스는 과제 제외라 미구현
- **체력바 색 (2026-07-05)**: 몬스터 = 빨강 고정, 플레이어 = 초록 고정. **그라데이션/색 변화 없음**
- **웨이브 = 행 컨베이어 (2026-07-07 재관찰로 정정)**: ~~전멸해야 다음 웨이브~~ → 시작에 5줄 일괄, 이후 **시간 간격으로 한 줄씩** 계속 스폰. **빈 행도 패턴에 포함** (그룹 사이 간격의 정체). 총 행 소진 + 전멸 = 클리어. [가정: 전멸 여부와 무관하게 시간 페이스 유지 — 재관찰 필요]. 구성은 랜덤 아닌 고정
- 진행도 % = 정해진 유닛 수 중 처치 비율. 마지막 보스는 과제 제외
- **3택지 스킬 선택 UI가 뜨면 게임 정지 (timeScale)** — 몬스터 하강도 멈춤 (2026-07-04)

## 스킬 동작 (2026-07-05 유저 관찰 — 수치는 기획서 PDF가 SSOT, 여기는 "거동"만)

- **파이어볼**: 타격 시 화상 부여, 지속시간 동안 **매초 중첩당 피해** (PDF Lv1: 4초, 최대 3중첩, 중첩당 초당 8)
- **아이스볼**: 확률 냉동 — 냉동 중 **이동속도 감소 + 받는 피해 증가** (PDF Lv1: 30%, 5초, 10%/10%)
- **레이저볼**: 타격 시 **같은 행의 모든 적**에게 피해 (PDF Lv1: 7)
- **고스트볼**: 적을 **관통하며 지나가는 모든 적에게 데미지**. 벽/바닥 반사는 유지 — **바닥에 부딪히면 회수** (노멀볼과 동일한 회수 경로)
- **클러스터볼**: 타격 시 확률로 **파란 파편볼 생성**, 파편볼이 충돌하면 피해 (PDF Lv1: 40%, 10) — 파편은 자동 발사체 느낌
- **발사 모델 = 볼 인벤토리 (2026-07-06 재관찰로 확정 — 로테이션/연사 모델 폐기)**: 보유 볼 각각이 실제 개체 — 필드에 나가 있거나 발사 대기. **먼저 회수된 볼부터 발사 간격을 두고 재발사** (쿨다운이 아니라 회수가 발사를 만든다). 시작 = 노멀볼 4발(필드에 3발 튕기고 1발 회수→재발사 순환 관찰). 액티브 획득 = 볼 1개 추가
- **노멀볼 채움 카드 (2026-07-06 유저 확정)**: 드래프트 후보가 3장 미만이면 빈 자리에 "노멀볼" 카드, 전부 소진되면 노멀볼 카드만. 획득 = **노멀볼 레벨 +1 그리고 개수 +1** — 전투 정보 창의 ◆xN이 이 레벨 (Lv1 = 시작 4발)
- **밸런스 메모**: 현재 몬스터 HP/레벨업 필요 킬이 너무 낮아 연속 레벨업 폭주 — 수치 튜닝 필요 (기획서 수치와 대조)
- 미관찰 [가정 처리]: 파편볼의 수명/반사 여부, 전면/후면 타격(단검 패시브)의 판정 기준

## 인게임 HUD (2026-07-06 스크린샷 확보)

- 좌상: 배속(▶)·Auto 버튼 → **구현 제외 (유저 결정)**
- 중상: 스테이지명 "N. 이름" + 바로 아래 얇은 **진행도 바** (검정 트랙+빨강 fill, % 텍스트, 오른쪽 끝 빨간 보스 아이콘)
- 우상: **일시정지 버튼** (원형, ⏸)
- 그 아래 전폭: **레벨(킬) 게이지** — 주황/불꽃 fill 바 + 오른쪽 끝 "문(door) 배지"에 현재 레벨 숫자
- 하단: 슈터 아래 초록 알약형 HP 바 + 흰 숫자 (우리 구현과 유사)
- 조준선: 점선(도트 나열) + 끝에 타겟 서클 (우리는 실선 라인렌더러 — 폴리시 여지)
- 데미지 플로터: 흰색 숫자, 타격 지점 위

## 전투 정보 창 (2026-07-07 스크린샷 #57 상세 확보)

- 구성: "전투 정보" 배너 / Stage N (Normal) / **진행도 바(46%, 빨강+보스 아이콘)** / 볼별 행 리스트(스크롤) / "터치하여 닫기"
- 행 구성: 아이콘 + **◆xN(스킬 레벨 — 클러스터 x2 확인)** / 이름 / **누적 데미지(콤마 포맷 2,209)** / **최대 대비 비율 빨간 바** / DPS
- **패시브도 행에 있음**: "마지막 성냥 DPS" 행 존재 = **부가 피해(폭발 등)도 소스별로 집계**된다는 증거
- 노멀 볼도 항상 행 존재 (누적 최다 — 바 가득) / 이름 표기 "레이저 볼(가로)" — 방향 명시
- DPS = 누적 ÷ 전투 경과시간으로 추정 (정지 시간 제외된 느낌)

- **레이저볼은 무한 반사 (2026-07-07 재확인)** — 히트마다 행 피해 발동이 원작 사양 (1회 회수로 바꿨다가 재관찰로 복구)

## 일시정지 화면 (2026-07-06 스크린샷 확보)

- "일시정지" 타이틀 + "Stage N (Normal)" + 통계 아이콘
- **Active Skill 슬롯 4칸(빨간 틀) / Passive Skill 슬롯 2칸(청록 틀)** — 보유 스킬 아이콘 표시. 우리 MaxActive=4/MaxPassive=2와 일치 (원작 검증)
- "현재 스테이지 드랍" 보상 그리드 (골드/스크롤) → 메타 경제, 스코프 아웃
- 하단: [홈] [이어하기] [설정] — 홈/설정은 아웃게임 영역이라 스코프 아웃, 이어하기만

## 결과 화면 흐름 (2026-07-06 스크린샷 확보)

패배 시 2단계: ①**부활 팝업** — 죽은 캐릭터(X눈, 옆으로 쓰러지는 죽음 모션 후) + "패배..." 배너 + "부활 후 이어하시겠습니까?" + 일일 부활 횟수(5/5) + [취소]/[부활(광고)] → ②**결과 화면(Stage Fail)** — 별 3개(빈), "Stage Fail" 배너 + "Stage N (Normal)", **진행도 % 바**, 획득 보상 그리드(골드/스킬 스크롤/아이템), [돌아가기]/[2배 획득(광고)]
승리: 같은 레이아웃 — 별 3개(금색 채움) + 왕관 연출, "Stage Clear" 배너, **"남은 체력 %" 바(초록)** ← Fail의 진행도 자리에 남은 체력이 옴. 보상 그리드/버튼 동일
→ 과제 스코프 판단: 광고 부활·보상 정산·2배 획득은 메타 경제(아웃게임) 영역 — 기획서 5항 요구는 "성공/실패 팝업 + 재시작"뿐
